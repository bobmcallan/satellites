Follow agile smallest-change delivery. When a story's scope proves larger or riskier than expected, identify the smallest independently-valuable change that ships now.

Use AC amendments to refine acceptance criteria that don't match code reality. File follow-up stories for remaining scope with explicit AC coverage. Close the current story as partially delivered, referencing follow-up story IDs. Never push through pressure-delivery style -- if the risk is ambiguous, escalate to the user.

The pipeline supports iterative delivery: ship what's proven, split what's risky, close the loop with follow-ups.
